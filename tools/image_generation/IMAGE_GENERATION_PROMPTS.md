## Implementation Guide

### Naming Conventions
Use consistent naming for generated assets:
```
Format: spiritatlas_[category]_[name]_[variant]_[density].png

Examples:
- spiritatlas_icon_app_primary_xxxhdpi.png
- spiritatlas_bg_home_cosmic_1080x1920.png
- spiritatlas_button_primary_gradient_360x120.png
- spiritatlas_avatar_masculine_default_512.png
- spiritatlas_symbol_chakra_root_512.png
```

### Android Resource Organization
```
res/
├── drawable/                    # Vector and XML drawables
├── drawable-mdpi/              # 48dp = 48px
├── drawable-hdpi/              # 48dp = 72px
├── drawable-xhdpi/             # 48dp = 96px
├── drawable-xxhdpi/            # 48dp = 144px
├── drawable-xxxhdpi/           # 48dp = 192px
└── mipmap-*/                   # App icons only
```

### Density Multipliers
- **mdpi (baseline):** 1x
- **hdpi:** 1.5x
- **xhdpi:** 2x
- **xxhdpi:** 3x
- **xxxhdpi:** 4x

### File Format Guidelines
- **Icons & Logos:** PNG with transparency
- **Backgrounds:** JPG (if no transparency) or PNG
- **Buttons & UI Elements:** PNG with transparency or 9-patch
- **Symbols & Badges:** PNG with transparency

### Optimization
- Run all PNGs through TinyPNG or similar compressor
- Target: <100KB for icons, <500KB for backgrounds
- Use WebP format for Android 4.3+ for better compression

### Color Accuracy
When generating images, include exact hex codes in prompts:
- Reference the Color.kt file for exact values
- Request "exact hex color #6B21A8" in prompts
- Verify generated colors match palette (use color picker)

### Quality Checklist
✅ Resolution matches requirements
✅ Colors match theme palette
✅ Transparent backgrounds where specified
✅ Sacred geometry is mathematically accurate
✅ Spiritual symbols are culturally authentic
✅ File sizes are optimized
✅ Naming convention followed
✅ All required densities generated
